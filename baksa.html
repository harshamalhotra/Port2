BAKSA

Baksa is a SaaS (Software as a Service) web application which allows people to collaborate with others in collecting and storing simple documents, images, and links to websites.

DELIVARABLE

Responsive Web App

SKILLS AND RESPONSIBILITY

UX Research, UX Design, Brand Identity

TOOLS AND SOFTWARE

Balsamiq, Sketch, Illustrator, Photoshop, Invision

THE CHALLENGE

A SIMPLE, RESPONSIVE WEB EXPERIENCE
FOR VARIETY OF CONTENT

There are many applications in the market which allows the users to store information. The main problem lies in their complexity. The challenge was to create an application which was simple and functional while making sure that the user experience is joyful.

SWOT ANALYSIS IN DETAIL

RESEARCH

SURVEY FINDINGS

1. Since many people were using different devices to access content, the app needed to be responsive.

2. Available storage in free version was important.

3. Current SaaS web solutions were complex and needed simplicity.

4. Users wanted an auto save facility while writing to avoid information loss.
ANALYSIS OF THE RESULTS

PERSONA

UNDERSTANDING POTENTIAL USERS

Based on the surveys, some power users cared about writing documents whereas most others cared about storing images. So I crafted user personas based on my research to ensure that I was designing for all the potential users.
PERSONAS IN DETAIL

IDENTITY AND BRANDING

DEFINING BRAND LANGUAGE

Baksa is a product which like a box can store information. To reinforce this idea, the logo was designed in the shape of a box. The fonts Garamond and Helvetica Neue were chosen to give the typography a blend of class and simplicity.

INFORMATION ARCHITECTURE

USER STORIES AND USER FLOWS

Based on business requirements and the user research, I determined a list of user stories that I needed to design for. I then created user flows for these stories.
USER FLOWS IN DETAIL

WIREFRAMES

EXECUTION ON THE SCREENS 

While sketching the hero page of Baksa, I created four versions of the hero page and conducted a preference test with 10 users. I then finally created the low fidelity wireframes using Balsamiq. After getting a fair idea about the structure, I then designed the medium fidelity wireframes in gray scale. After receiving a positive feedback on a usability test conducted on Peek, I created the high fidelity wireframes by adding color and content to the medium fidelity wireframes.

DEVELOPMENT

CODING WITH HTML AND CSS

I deployed baksa on Gitbhub after coding it using HTML and CSS. With the of use media queries, I made it responsive. This was a challenge and I eventually learnt Chrome's testing tools while figuring out how to test the design in different screen sizes.

USABILITY TESTING

MAKING IT MORE USABLE

I asked three people how they felt about baksa’s design. They really appreciated the use of bold colors and thought the site looked easy to use. They especially liked the display of the items on the dashboard.The users thought that they would like to have more functionality in the product. They also wanted the sign out option to be more visible.

LESSONS LEARNT

Baksa was a great learning experience for me. I used design tools like Balsamiq, Sketch, Photoshop, and Illustrator for the very first time. I employed various research methods and fused my results into the design process. I also learnt to write HTML and CSS along with the media queries. In addition, I became more comfortable the terminal and also learnt to version control my code using git and github. All in all, I learnt a lot while creating baksa and the confidence that I developed from working on baksa helped me hone my skills in future projects.
